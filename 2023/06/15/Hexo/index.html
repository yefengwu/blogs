Create Github Repos

Create two repos in github, one public repos named blogs , one private repo named blosg-source .

The main branch of a private repository is used to store the Hexo source code.
The gh-pages branch of a public repository is used to store the generated Hexo HTML files.

When changes are pushed to the main branch of the private repository, GitHub Actions will automatically be triggered. It will then compile the Hexo source code and deploy the generated HTML files to the gh-pages branch of the public repository.

Create Github Token

Create a GitHub Token with repo and workflow permissions.

Once a token is generated and displayed, it won’t be shown again for security reasons.

Create Repository Secret

Add the token generated earlier to the ‘secrets’ section of the private repository and rename this newly added ‘secret’ as ‘HEXO_DEPLOY’.

Steps：private repository -> settings -> Secrets -> New repository secret。

The newly created secret ‘HEXO_DEPLOY’ needs to be used in the Actions configuration file, and it needs to be consistent with the configuration file!

Automatic deployment triggering process

  1. Modify your Hexo blog source code (e.g. add articles, modify articles, change themes, modify theme configuration files, etc.).
  2. Push your modified Hexo project content (only commit the changed part) to the ‘main’ branch of the GitHub private repository (e.g., blogs-source).
  3. GitHub Actions detects that content ‘push’ is coming in from the ‘main’ branch, automatically executes the action configuration file’s commands, compiles the Hexo project into a static page, and deploys it to the ‘gh-pages’ branch of the public repository.
  4. In the private repository, you can view the action you have configured in Actions.

Install Plugin

  1. Install a plugin called “Github Publisher”.
  2. Configure the repository name as “blogs-source”, set the Github username as <Username>, and specify the main branch as “main” in the Github Configuration section.
  3. Configure the default folder to be source/_posts in the Upload configuration tab.

Now that all the tasks have been completed, you may proceed with invoking the “publish files” command to share the note with Hexoe
